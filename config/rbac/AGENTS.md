# rbac

## Purpose
ClusterRoles/Roles and bindings for the operator's ServiceAccount, the metrics endpoint, leader election, and the user-facing editor/viewer roles.

## Key Files

| File | Description |
|------|-------------|
| `role.yaml` | Generated from `+kubebuilder:rbac:` markers on the reconciler. Regenerated by `make manifests`. |
| `role_binding.yaml` | Binds `role.yaml` to the manager ServiceAccount |
| `service_account.yaml` | The manager's ServiceAccount |
| `leader_election_role.yaml` / `..._binding.yaml` | Coordinated lease for HA |
| `metrics_auth_role.yaml` / `..._binding.yaml` | TokenReview/SubjectAccessReview perms for the metrics endpoint |
| `metrics_reader_role.yaml` | ClusterRole granting read access to the metrics endpoint (bind separately) |
| `migration_editor_role.yaml` / `migration_viewer_role.yaml` | User-facing aggregated roles |
| `kustomization.yaml` | Kustomize wiring |

## For AI Agents

### Working In This Directory
- Do not hand-edit `role.yaml`. To grant the reconciler new resource access, add or change a `+kubebuilder:rbac:` marker in `internal/controller/migration_controller.go` and run `make manifests`.
