# crd

## Purpose
Kustomize base for the `Migration` CRD. The CRD itself is generated by `controller-gen`; patches enable optional conversion-webhook and CA-injection wiring.

## Key Files

| File | Description |
|------|-------------|
| `kustomization.yaml` | Kustomize entry; pulls `bases/flyway.davidkarlsen.com_migrations.yaml` and the patches |
| `kustomizeconfig.yaml` | Tells kustomize which fields hold name references in CRDs (so renames propagate) |

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `bases/` | Generated CRD YAML — regenerated by `make manifests` (see `bases/AGENTS.md`) |
| `patches/` | Optional CA-injection / webhook patches commented out by default (see `patches/AGENTS.md`) |

## For AI Agents

### Working In This Directory
- Never hand-edit `bases/flyway.davidkarlsen.com_migrations.yaml` — always go through API types + `make manifests`.
- The same generated CRD is also copied to `config/helm-chart/flyway-operator/crds/`; `make manifests` (or the helm-chart packaging step) keeps them in sync.
