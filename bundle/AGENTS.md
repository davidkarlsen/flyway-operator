# bundle

## Purpose
OLM bundle output. Generated by `make bundle` from `config/manifests/`, `config/crd/`, `config/default/`, and `config/scorecard/`. The image is built with `bundle.Dockerfile` and published to OperatorHub-compatible catalogs (see `Dockerfile.catalog`, `catalog-template.yaml`).

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `manifests/` | CSV + CRD + RBAC + Service + ServiceMonitor included in the bundle (see `manifests/AGENTS.md`) |
| `metadata/` | Bundle annotations (channels, mediatype, package name) (see `metadata/AGENTS.md`) |
| `tests/` | Scorecard test config baked into the bundle (see `tests/AGENTS.md`) |

## For AI Agents

### Working In This Directory
- These files are regenerated by `make bundle` — hand-edits are typically clobbered. Edit the source under `config/manifests/bases/` (CSV seed) or `config/scorecard/` (test config) instead, then regenerate.
- Bundle version comes from `VERSION` in the `Makefile` (or `make bundle VERSION=x.y.z`).
