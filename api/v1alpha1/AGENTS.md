# v1alpha1

## Purpose
CRD types for the `flyway.davidkarlsen.com/v1alpha1` API group. Defines the `Migration` resource (spec, status, helpers) and registers it with the runtime scheme.

## Key Files

| File | Description |
|------|-------------|
| `migration_types.go` | `Migration`, `MigrationSpec`, `MigrationStatus`, `Database`, `FlywayConfiguration`, `MigrationSource` types and helper methods (`IsPaused`, `GetConditions`, `SetConditions`, `GenerationAsString`, `GetCredentials`, `GetPlaceholdersAsEnvVars`). Also exports the `Prefix`, `Generation`, and (private) `paused` annotation constants. |
| `groupversion_info.go` | `GroupVersion`, `SchemeBuilder` (built via `runtime.NewSchemeBuilder(addKnownTypes)`), `AddToScheme`. The `addKnownTypes` function registers `&Migration{}` and `&MigrationList{}`. |
| `zz_generated.deepcopy.go` | Generated by `controller-gen`. Do not hand-edit — run `make generate`. |

## For AI Agents

### Working In This Directory
- Use kubebuilder validation markers on every spec field: `+kubebuilder:validation:Required|Optional|Pattern|Enum`, `+kubebuilder:default`. The `JdbcUrl` field already pins `^jdbc:.*` — keep that pattern.
- After editing types, run `make manifests generate` to refresh `config/crd/bases/...` and `zz_generated.deepcopy.go`. Update `config/samples/flyway_v1alpha1_migration.yaml` if the public shape changed.
- All operator-specific annotations must be built from the `Prefix` constant. Adding a new annotation? Add a typed const next to `Generation` / `paused`.
- Scheme registration is done in `addKnownTypes` — do **not** reintroduce an `init()` that calls `SchemeBuilder.Register(...)`, and do **not** switch back to the deprecated `sigs.k8s.io/controller-runtime/pkg/scheme.Builder` (`SA1019`). The exported GroupVersion is `flywayv1alpha1.GroupVersion`; there is no `SchemeBuilder.GroupVersion` field on `runtime.SchemeBuilder`.

### Testing Requirements
- This package has no `*_test.go`; coverage comes from `internal/controller/` tests via `make test`.

### Common Patterns
- Status is exclusively `[]metav1.Condition` with `patchMergeKey:"type"` and `listType=map`.
- Helpers on `*Migration` (e.g. `IsPaused`, `GetConditions`) satisfy interfaces from `redhat-cop/operator-utils` so the reconciler can use its generic helpers.
- `MigrationSource.GetPlaceholdersAsEnvVars()` translates user placeholders into `FLYWAY_PLACEHOLDERS_<KEY>` env vars consumed by the Job spec builder.
